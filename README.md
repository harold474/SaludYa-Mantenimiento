# SaludYa - Mantenimiento de Software

Este repositorio contiene la estructura unificada del proyecto **SaludYa** (Frontend y Backend), adaptado para el módulo de Mantenimiento de Software mediante un flujo de trabajo basado en ramas (*Git Feature Branch Workflow*).

## 📋 Historias de Usuario Implementadas

* **HU01:** Como paciente, quiero visualizar los horarios médicos disponibles para poder agendar una cita rápidamente.
* **HU02:** Como médico, quiero acceder a mi panel para ver las citas programadas de mi jornada.
* **HU03:** Como administrador del sistema, necesito registrar nuevas especialidades médicas en la base de datos para ampliar el portafolio de servicios.
* **HU04:** Como paciente, quiero tener un botón para cancelar mi cita con 24 horas de antelación en caso de un imprevisto.
* **HU05:** Como desarrollador, necesito implementar un archivo `.gitignore` para evitar que las credenciales de conexión a la base de datos sean expuestas en el repositorio público.
